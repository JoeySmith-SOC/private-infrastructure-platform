# Screenshots

## Purpose

Store sanitized screenshots that support documentation, validation evidence, and portfolio summaries.

## Folder Structure

| Path | Purpose |
|---|---|
| `screenshots/architecture/` | Platform-wide captures |
| `screenshots/networking/` | Switch, routing, and addressing evidence |
| `screenshots/firewall/` | OPNsense and VPN screenshots |
| `screenshots/proxmox/` | Node and cluster screenshots |
| `screenshots/storage/` | TrueNAS and backup evidence |
| `screenshots/monitoring/` | Grafana and Prometheus dashboards |
| `screenshots/security/` | Wazuh and security evidence |
| `screenshots/kubernetes/` | Future cluster screenshots |
| `screenshots/ot-lab/` | Future OT/ICS lab evidence |

## Standards

- Redact secrets, public IPs, serials, and usernames if needed.
- Use descriptive filenames with dates when screenshots capture milestone evidence.
- Reference screenshots directly from the documents that depend on them.
