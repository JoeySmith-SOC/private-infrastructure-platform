# Planning Folder

## Purpose

This folder holds strategic documents that explain where the platform is going, why it exists, how phases are sequenced, and how the project should be presented externally.

## What Belongs Here

- Vision and success criteria
- Roadmaps and phase sequencing
- Portfolio-facing summaries
- Planning indices, decision workflows, and procurement planning support

## Folder Structure

| Path | Purpose |
|---|---|
| `planning/project-vision.md` | Mission, scope, and design intent |
| `planning/expansion-roadmap.md` | Phase-by-phase growth path |
| `planning/portfolio-blog-ready-summaries.md` | Resume, blog, and case-study summaries |
| `planning/decisions/` | Supporting planning notes and future ADR intake |
| `planning/phases/` | Phase-level planning artifacts |
| `planning/procurement/` | Procurement planning support |
| `planning/roadmaps/` | Additional roadmap variants or milestone views |

## Key Questions

- What is the platform trying to become over time?
- What should be built next and why?
- How is project progress communicated to employers, clients, or collaborators?
- Which planning artifacts are authoritative versus supporting?
