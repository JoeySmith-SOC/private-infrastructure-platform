# Runbooks

## Purpose

This folder contains operational procedures for routine maintenance, incident response, recovery, and validation.

## Structure

| Path | Purpose |
|---|---|
| `runbooks/maintenance-runbook.md` | Planned operations, patching, validation, and upkeep |
| `runbooks/disaster-recovery-runbook.md` | Major outage and bare-metal recovery procedures |
| `runbooks/operations/` | Future day-2 operating procedures |
| `runbooks/incidents/` | Future incident-specific playbooks |
| `runbooks/recovery/` | Future service or component recovery drills |
| `runbooks/validation/` | Future post-change and DR test checklists |

## Key Questions

- What procedure should be followed during normal maintenance?
- What is the first response during infrastructure failure?
- What evidence must be captured during tests and incidents?
- Which runbooks must be updated after each platform change?
